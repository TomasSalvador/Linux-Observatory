Linux Observatory is a small observability platform built to explore Linux systems programming, kernel development, and low-level software engineering. The project collects and exposes system metrics such as CPU, memory, process, and network usage through a multithreaded monitoring daemon, a command-line client, and a custom Linux kernel module. It serves as a practical demonstration of Linux internals, concurrency, networking, debugging, automation, and software packaging.


OR

I created Linux Observatory to gain practical experience with the areas most commonly found in Linux systems and kernel engineering roles. Rather than focusing on a single feature, the project was designed as a learning platform covering Linux internals, concurrency, networking, debugging, automation, packaging, and kernel development.

The goal is not to replace existing monitoring tools, but to understand how such tools are built and how software interacts with the operating system at different layers—from kernel modules and /proc interfaces to user-space services and command-line applications.
